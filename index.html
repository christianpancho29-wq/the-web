Observations & analysis
Strength: Polyalphabetic layer defeats single-letter frequency analysis.
Strength: Rail-fence transposition breaks bigram and trigram patterns.
Strength: Chunk reversal is key-length-dependent — different keys → structurally different ciphertexts.
Strength: Hex encoding ensures output is always printable ASCII.
Limitation: Key stream period = key length. Short keys weaken long-message security.
Limitation: Symmetric — same key for encrypt and decrypt. Key must be shared securely.
Limitation: Unicode chars outside the 94-char ALPHABET pass through unencrypted.
Limitation: Educational-grade only — not suitable for protecting sensitive real-world data.
Algorithm documentation

Overview

PolyShift is a symmetric multi-layer classical cipher combining four distinct transformation techniques designed to resist simple frequency analysis and pattern recognition.

Encryption pipeline

  • 1
    Key Stream Derivation — ks[i] = charCode(key[i mod keyLen]). Repeating numeric stream from passphrase.
  • 2
    Keystream Substitution — ALPHABET[(index + ks[i]) mod 94]. Polyalphabetic — same char maps differently per position.
  • 3
    Caesar Shift — Shifts A–Z and a–z by offset. Adds a second independent key parameter.
  • 4
    Rail-Fence Transposition — 3-rail zigzag write, row-by-row read. Separates adjacent characters positionally.
  • 5
    Chunk Reversal — Blocks of size=key.length are each reversed. Key-dependent structure.
  • 6
    Hex Encoding — Each char → 2-digit hex. Output is always [0-9a-f], always printable.

Decryption (exact reverse)

  • 1
    Hex Decode
  • 2
    Un-Reverse Chunks
  • 3
    Inverse Rail-Fence
  • 4
    Reverse Caesar Shift
  • 5
    Reverse Keystream Substitution
